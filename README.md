        <h1 dir="ltr">Submission Bookshelf API</h1><p dir="ltr">Selamat! Akhirnya Anda telah sampai di penghujung pembelajaran. Anda telah mempelajari:</p><ul><li dir="ltr">Istilah <em>back-end</em>, <em>front-end</em>, <em>server</em>, <em>client</em>, <em>web service</em>, REST API serta belajar cara berkomunikasi antara <em>client </em>dengan <em>server </em>menggunakan cURL.</li><li dir="ltr">Dasar-dasar Node.js dalam pengembangan Back-End seperti <em>global object</em>, <em>process</em>, <em>modularization</em>, <em>event</em>, dan yang lainnya.</li><li dir="ltr">Membangun <em>web server</em> menggunakan Node.js <em>native </em>maupun Hapi Framework. Selain itu, Anda sudah membuat RESTful API sederhana yang mendukung fungsionalitas aplikasi catatan.</li><li dir="ltr">Melakukan <em>deploy </em>RESTful API pada Amazon EC2.</li><li dir="ltr"><p dir="ltr">Pengujian RESTful API secara otomatis menggunakan Postman.</p></li></ul><p dir="ltr">Tentu Anda juga sudah mengerjakan seluruh latihan yang diberikan pada kelas ini.</p><p dir="ltr">Untuk bisa lulus dan mendapatkan sertifikat dari akademi ini, Anda harus mengerjakan tugas yakni membuat proyek <strong>Bookshelf API</strong> sesuai kriteria lengkap di bawah ini. Tim Reviewer akan memeriksa pekerjaan Anda dan memberikan reviu pada proyek yang Anda buat.</p><p dir="ltr"><br /></p><h2 dir="ltr">Kriteria Bookshelf API</h2><p dir="ltr">Terdapat 5 kriteria utama yang harus Anda penuhi dalam membuat proyek Bookshelf API.</p><h3 dir="ltr">Kriteria 1 : API dapat menyimpan buku</h3><p dir="ltr">API yang Anda buat harus dapat menyimpan buku melalui <em>route</em>:</p><ul><li dir="ltr">Method : <strong>POST</strong></li><li dir="ltr">URL : <strong>/books</strong></li><li dir="ltr"><p dir="ltr">Body Request:</p><pre dir="ltr">{
    "name": string,
    "year": number,
    "author": string,
    "summary": string,
    "publisher": string,
    "pageCount": number,
    "readPage": number,
    "reading": boolean
}</pre></li></ul><p dir="ltr">Objek buku yang disimpan pada <em>server </em>harus memiliki struktur seperti contoh di bawah ini:</p><pre dir="ltr">{
    <strong>"id": "Qbax5Oy7L8WKf74l",</strong>
    "name": "Buku A",
    "year": 2010,
    "author": "John Doe",
    "summary": "Lorem ipsum dolor sit amet",
    "publisher": "Dicoding Indonesia",
    "pageCount": 100,
    "readPage": 25,
   <strong> "finished": false,</strong>
    "reading": false,
  <strong>  "insertedAt": "2021-03-04T09:11:44.598Z",
    "updatedAt": "2021-03-04T09:11:44.598Z"</strong>
}</pre><p dir="ltr">Properti yang ditebalkan diolah dan didapatkan di sisi <em>server</em>. Berikut penjelasannya:</p><ul><li dir="ltr"><span style="padding:2px 4px;color:rgb(199,37,78);background-color:rgb(249,242,244);">id</span> : nilai <span style="padding:2px 4px;color:rgb(199,37,78);background-color:rgb(249,242,244);">id</span> haruslah unik. Untuk membuat nilai unik, Anda bisa memanfaatkan <a href="https://www.npmjs.com/package/nanoid" target="_blank" rel="noreferrer noopener">nanoid</a>.</li><li dir="ltr"><span style="padding:2px 4px;color:rgb(199,37,78);background-color:rgb(249,242,244);">finished</span> : merupakan properti <em>boolean </em>yang menjelaskan apakah buku telah selesai dibaca atau belum. Nilai <span style="padding:2px 4px;color:rgb(199,37,78);background-color:rgb(249,242,244);">finished</span> didapatkan dari observasi <span style="padding:2px 4px;color:rgb(199,37,78);background-color:rgb(249,242,244);">pageCount === readPage</span>.</li><li dir="ltr"><span style="padding:2px 4px;color:rgb(199,37,78);background-color:rgb(249,242,244);">insertedAt</span> : merupakan properti yang menampung tanggal dimasukkannya buku. Anda bisa gunakan <span style="padding:2px 4px;color:rgb(199,37,78);background-color:rgb(249,242,244);">new Date().toISOString()</span> untuk menghasilkan nilainya.</li><li dir="ltr"><p dir="ltr"><span style="padding:2px 4px;color:rgb(199,37,78);background-color:rgb(249,242,244);">updatedAt</span> : merupakan properti yang menampung tanggal diperbarui buku. Ketika buku baru dimasukkan, berikan nilai properti ini sama dengan <span style="padding:2px 4px;color:rgb(199,37,78);background-color:rgb(249,242,244);">insertedAt</span>.</p></li></ul><p dir="ltr">Server harus merespons <strong>gagal </strong>bila:</p><ul><li dir="ltr">Client tidak melampirkan properti <span style="padding:2px 4px;color:rgb(199,37,78);background-color:rgb(249,242,244);">name</span>pada <em>request body</em>. Bila hal ini terjadi, maka <em>server </em>akan merespons dengan:<ul><li dir="ltr">Status Code : <strong>400</strong></li><li dir="ltr"><p dir="ltr">Response Body:</p><pre dir="ltr">{
    "status": "fail",
    "message": "Gagal menambahkan buku. Mohon isi nama buku"
}</pre></li></ul></li><li dir="ltr">Client melampirkan nilai properti <span style="padding:2px 4px;color:rgb(199,37,78);background-color:rgb(249,242,244);">readPage</span> yang lebih besar dari nilai properti <span style="padding:2px 4px;color:rgb(199,37,78);background-color:rgb(249,242,244);">pageCount</span>. Bila hal ini terjadi, maka <em>server </em>akan merespons dengan:<ul><li>Status Code : <strong>400</strong></li><li><p>Response Body:</p><pre>{
    "status": "fail",
    "message": "Gagal menambahkan buku. readPage tidak boleh lebih besar dari pageCount"
}</pre></li></ul></li><li dir="ltr">Server gagal memasukkan buku karena alasan umum (<em>generic error</em>). Bila hal ini terjadi, maka <em>server </em>akan merespons dengan:<ul><li dir="ltr">Status Code : <strong>500</strong></li><li dir="ltr"><p dir="ltr">Response Body:</p><pre dir="ltr">{
    "status": "error",
    "message": "Buku gagal ditambahkan"
}</pre></li></ul></li></ul><p dir="ltr">Bila buku <strong>berhasil </strong>dimasukkan, <em>server </em>harus mengembalikan respons dengan:</p><ul><li dir="ltr">Status Code : <strong>201</strong></li><li dir="ltr"><p dir="ltr">Response Body:</p><pre dir="ltr">{
    "status": "success",
    "message": "Buku berhasil ditambahkan",
    "data": {
        "bookId": "1L7ZtDUFeGs7VlEt"
    }
}</pre></li></ul><p><br /></p><h3 dir="ltr">Kriteria 2 : API dapat menampilkan seluruh buku</h3><p dir="ltr">API yang Anda buat harus dapat menampilkan seluruh buku yang disimpan melalui <em>route</em>:</p><ul><li dir="ltr">Method : <strong>GET</strong></li><li dir="ltr"><p dir="ltr">URL: <strong>/books</strong></p></li></ul><p dir="ltr">Server harus mengembalikan respons dengan:</p><ul><li dir="ltr">Status Code : <strong>200</strong></li><li dir="ltr"><p dir="ltr">Response Body:</p><pre dir="ltr">{
    "status": "success",
    "data": {
        "books": [
            {
                "id": "Qbax5Oy7L8WKf74l",
                "name": "Buku A",
                "publisher": "Dicoding Indonesia"
            },
            {
                "id": "1L7ZtDUFeGs7VlEt",
                "name": "Buku B",
                "publisher": "Dicoding Indonesia"
            },
            {
                "id": "K8DZbfI-t3LrY7lD",
                "name": "Buku C",
                "publisher": "Dicoding Indonesia"
            }
        ]
    }
}</pre></li></ul><p>Jika <strong>belum </strong>terdapat buku yang dimasukkan, <em>server </em>bisa merespons dengan <em>array </em><span style="padding:2px 4px;color:rgb(199,37,78);background-color:rgb(249,242,244);">books</span> kosong.</p><pre>{
    "status": "success",
    "data": {
        "books": []
    }
}</pre><p><br /></p><h3 dir="ltr">Kriteria 3 : API dapat menampilkan detail buku</h3><p dir="ltr">API yang Anda buat harus dapat menampilkan seluruh buku yang disimpan melalui <em>route</em>:</p><ul><li dir="ltr">Method : <strong>GET</strong></li><li dir="ltr"><p dir="ltr">URL: <strong>/books/{bookId}</strong></p></li></ul><p dir="ltr">Bila buku dengan <span style="padding:2px 4px;color:rgb(199,37,78);background-color:rgb(249,242,244);">id</span><strong> </strong>yang dilampirkan oleh <em>client </em>tidak ditemukan, maka <em>server </em>harus mengembalikan respons dengan:</p><ul><li dir="ltr">Status Code : <strong>404</strong></li><li dir="ltr"><p dir="ltr">Response Body:</p><pre dir="ltr">{
    "status": "fail",
    "message": "Buku tidak ditemukan"
}</pre></li></ul><p dir="ltr">Bila buku dengan <span style="padding:2px 4px;color:rgb(199,37,78);background-color:rgb(249,242,244);">id</span><strong> </strong>yang dilampirkan <strong>ditemukan</strong>, maka <em>server </em>harus mengembalikan respons dengan:</p><ul><li dir="ltr">Status Code : <strong>200</strong></li><li dir="ltr"><p dir="ltr">Response Body:</p><pre dir="ltr">{
    "status": "success",
    "data": {
        "book": {
            "id": "aWZBUW3JN_VBE-9I",
            "name": "Buku A Revisi",
            "year": 2011,
            "author": "Jane Doe",
            "summary": "Lorem Dolor sit Amet",
            "publisher": "Dicoding",
            "pageCount": 200,
            "readPage": 26,
            "finished": false,
            "reading": false,
            "insertedAt": "2021-03-05T06:14:28.930Z",
            "updatedAt": "2021-03-05T06:14:30.718Z"
        }
    }
}</pre></li></ul><p><br /></p><h3 dir="ltr">Kriteria 4 : API dapat mengubah data buku</h3><p dir="ltr">API yang Anda buat harus dapat mengubah data buku berdasarkan <span style="padding:2px 4px;color:rgb(199,37,78);background-color:rgb(249,242,244);">id</span> melalui <em>route</em>:</p><ul><li dir="ltr">Method : <strong>PUT</strong></li><li dir="ltr">URL : <strong>/books/{bookId}</strong></li><li dir="ltr"><p dir="ltr">Body Request:</p><pre dir="ltr">{
    "name": string,
    "year": number,
    "author": string,
    "summary": string,
    "publisher": string,
    "pageCount": number,
    "readPage": number,
    "reading": boolean
}</pre></li></ul><p dir="ltr">Server harus merespons <strong>gagal </strong>bila:</p><ul><li dir="ltr">Client tidak melampirkan properti <span style="padding:2px 4px;color:rgb(199,37,78);background-color:rgb(249,242,244);">name</span> pada <em>request body</em>. Bila hal ini terjadi, maka <em>server </em>akan merespons dengan:<ul><li dir="ltr">Status Code : <strong>400</strong></li><li dir="ltr"><p dir="ltr">Response Body:</p><pre>{
    "status": "fail",
    "message": "Gagal memperbarui buku. Mohon isi nama buku"
}</pre></li></ul></li><li dir="ltr">Client melampirkan nilai properti <span style="padding:2px 4px;color:rgb(199,37,78);background-color:rgb(249,242,244);">readPage</span> yang lebih besar dari nilai properti <span style="padding:2px 4px;color:rgb(199,37,78);background-color:rgb(249,242,244);">pageCount</span>. Bila hal ini terjadi, maka <em>server </em>akan merespons dengan:<ul><li dir="ltr">Status Code : <strong>400</strong></li><li dir="ltr"><p dir="ltr">Response Body:</p><pre>{
    "status": "fail",
    "message": "Gagal memperbarui buku. readPage tidak boleh lebih besar dari pageCount"
}</pre></li></ul></li><li dir="ltr"><span style="padding:2px 4px;color:rgb(199,37,78);background-color:rgb(249,242,244);">Id</span>yang dilampirkan oleh <em>client </em>tidak ditemukkan oleh <em>server</em>. Bila hal ini terjadi, maka <em>server </em>akan merespons dengan:<ul><li dir="ltr">Status Code : <strong>404</strong></li><li dir="ltr"><p dir="ltr">Response Body:</p><pre>{
    "status": "fail",
    "message": "Gagal memperbarui buku. Id tidak ditemukan"
}</pre></li></ul></li></ul><p dir="ltr">Bila buku <strong>berhasil diperbarui</strong>, <em>server </em>harus mengembalikan respons dengan:</p><ul><li dir="ltr">Status Code : <strong>200</strong></li><li dir="ltr"><p dir="ltr">Response Body:</p><pre dir="ltr">{
    "status": "success",
    "message": "Buku berhasil diperbarui"
}</pre></li></ul><p><br /></p><h3 dir="ltr">Kriteria 5 : API dapat menghapus buku</h3><p dir="ltr">API yang Anda buat harus dapat menghapus buku berdasarkan <span style="padding:2px 4px;color:rgb(199,37,78);background-color:rgb(249,242,244);">id</span> melalui <em>route </em>berikut:</p><ul><li dir="ltr">Method : <strong>DELETE</strong></li><li>URL: /books/{bookId}</li></ul><p dir="ltr">Bila <span style="padding:2px 4px;color:rgb(199,37,78);background-color:rgb(249,242,244);">id</span> yang dilampirkan tidak dimiliki oleh buku manapun, maka <em>server </em>harus mengembalikan respons berikut:</p><ul><li dir="ltr">Status Code : <strong>404</strong></li><li dir="ltr"><p dir="ltr">Response Body:</p><pre dir="ltr">{

    "status": "success",
    "message": "Buku berhasil dihapus"

Caranya, ekstrak<em> </em>berkas yang sudah diunduh hingga menghasilkan dua berkas file JSON.</p><p><img src="https://d17ivq9b7rppb3.cloudfront.net/original/academy/20210309105735c11efbd0f3d302ddab33746439713360.jpeg" class="fr-fic fr-dii" alt="20210309105735c11efbd0f3d302ddab33746439713360.jpeg" /></p><p dir="ltr">Kemudian pada aplikasi Postman, klik tombol <strong>import </strong>yang berada di atas panel kiri aplikasi Postman.</p><p><img src="https://d17ivq9b7rppb3.cloudfront.net/original/academy/20210404232830d775fe8e03382181a0ce401771b86e4d.png" class="fr-fic fr-dib fr-fil" alt="20210404232830d775fe8e03382181a0ce401771b86e4d.png" /></p><p>Kemudian klik tombol <strong>Upload Files</strong> untuk meng-<em>import</em> kedua berkas JSON hasil ekstraksi.</p><p><img src="https://d17ivq9b7rppb3.cloudfront.net/original/academy/202103091058148d67a5d026b67c1f2950e2745457ec6b.jpeg" class="fr-fic fr-dii" alt="202103091058148d67a5d026b67c1f2950e2745457ec6b.jpeg" /></p><p>Setelah itu, Bookshelf API Test Collection dan Environment akan tersedia pada Postman Anda.</p><table class="tab-code table table-striped" style="width:100%;"><tbody><tr><td class="tab-title">Collection</td></tr><tr><td class="tab-content"><img src="https://d17ivq9b7rppb3.cloudfront.net/original/academy/20210309105842cd9d8f20a00fb3897c8f6947d3a73cac.jpeg" class="fr-fic fr-dii" alt="20210309105842cd9d8f20a00fb3897c8f6947d3a73cac.jpeg" /></td></tr><tr><td class="tab-title">Environment</td></tr><tr><td class="tab-content"><img src="https://d17ivq9b7rppb3.cloudfront.net/original/academy/202103091058593958b895a74b159e1d4a9f6e2fbbc632.jpeg" class="fr-fic fr-dii" alt="202103091058593958b895a74b159e1d4a9f6e2fbbc632.jpeg" /></td></tr></tbody></table><p><br /></p><h2 dir="ltr">Kriteria Penilaian Submission</h2><p dir="ltr">Submission Anda akan dinilai oleh Reviewer guna menentukkan kelulusan Anda. Untuk lulus dari kelas ini, proyek Bookshelf API harus memenuhi seluruh pengujian otomatis pada Postman <em>request </em>yang bertanda <strong>[Mandatory]</strong>. Bila salah satu pengujiannya gagal, maka proyek Anda akan kami tolak.</p><p dir="ltr">Submission Anda akan dinilai oleh Reviewer dengan <strong>skala 1-5</strong>. Untuk mendapatkan nilai tinggi, silakan penuhi pengujian otomatis pada <em>request </em>yang bertanda <strong>[Optional]</strong>. Berikut detail dari fitur atau persyaratan opsional dari submission ini:</p><ul><li dir="ltr">Tambahkan fitur <em>query </em><em>parameters </em>pada <em>route </em><strong>GET /books </strong>(Mendapatkan seluruh buku).<ul><li><span style="padding:2px 4px;color:rgb(199,37,78);background-color:rgb(249,242,244);">?name</span> : Tampilkan seluruh buku yang mengandung nama berdasarkan nilai yang diberikan pada <em>query </em>ini. Contoh /books?name=dicoding, maka akan menampilkan daftar buku yang mengandung nama dicoding secara <strong><em>non-case sensitive  </em></strong>(tidak peduli besar dan kecil huruf).</li><li dir="ltr"><span style="padding:2px 4px;color:rgb(199,37,78);background-color:rgb(249,242,244);">?reading</span> : Bernilai 0 atau 1. Bila 0, maka tampilkan buku yang sedang tidak dibaca (reading: false). Bila 1, maka tampilkan buku yang sedang dibaca (reading: true). Selain itu, tampilkan buku baik sedang dibaca atau tidak.</li><li dir="ltr"><span style="padding:2px 4px;color:rgb(199,37,78);background-color:rgb(249,242,244);">?finished</span> : Bernilai 0 atau 1. Bila 0, maka tampilkan buku yang sudah belum selesai dibaca (finished: false). Bila 1, maka tampilkan buku yang sudah selesai dibaca (finished: true). Selain itu, tampilkan buku baik yang sudah selesai atau belum dibaca.</li></ul></li><li dir="ltr">Menerapkan <strong>CORS</strong> pada seluruh <em>resource </em>yang ada.</li><li dir="ltr"><p dir="ltr">Menggunakan ESLint dan salah satu <em>style</em> <em>guide </em>agar gaya penulisan kode JavaScript lebih konsisten.</p></li></ul><p dir="ltr">Berikut adalah detail penilaian submission:</p><ul><li dir="ltr"><strong>Bintang 1 </strong>: Semua ketentuan wajib terpenuhi, namun terdapat indikasi kecurangan dalam mengerjakan submission.</li><li dir="ltr"><strong>Bintang 2 </strong>: Semua ketentuan wajib terpenuhi, namun terdapat kekurangan pada penulisan kode. Seperti tidak menerapkan <em>modularization</em> atau gaya penulisan tidak konsisten.</li><li dir="ltr"><strong>Bintang 3 </strong>: Semua ketentuan wajib terpenuhi, namun tidak terdapat improvisasi atau persyaratan opsional yang dipenuhi.</li><li dir="ltr"><strong>Bintang 4 </strong>: Semua ketentuan wajib terpenuhi dan menerapkan minimal satu persyaratan opsional.</li><li dir="ltr"><p dir="ltr"><strong>Bintang 5 </strong>: Semua ketentuan wajib terpenuhi dan menerapkan seluruh persyaratan opsional yang ada.</p></li></ul><table class="table table-striped" style="width:100%;"><tbody><tr><td style="width:100%;background-color:rgb(216,251,250);"><span style="color:rgb(0,96,33);"><strong>Catatan: <br /></strong>Jika <em>submission Anda ditolak</em> maka <em>tidak ada penilaian</em>. Kriteria penilaian bintang di atas hanya berlaku <em>jika submission Anda lulus</em>.</span></td></tr></tbody></table><p><br /></p><h2 dir="ltr">Ketentuan Berkas Submission</h2><ul><li dir="ltr">Berkas submission yang dikirim merupakan folder proyek dari Bookshelf API dalam bentuk <strong>ZIP</strong>. </li><li dir="ltr">Pastikan di dalam folder proyek yang Anda kirim terdapat berkas <strong>package.json</strong>.</li><li dir="ltr"><p dir="ltr">Pastikan Anda hapus dulu berkas <strong>node_modules</strong> pada folder proyek sebelum mengkompresi dalam bentuk <strong>ZIP</strong>.</p></li></ul><p><br /></p><h2 dir="ltr">Submission Anda akan Ditolak bila</h2><ul><li dir="ltr">Kriteria wajib Bookshelf API tidak terpenuhi.</li><li dir="ltr">Ketentuan berkas submission tidak terpenuhi.</li><li dir="ltr">Proyek yang Anda kirim tidak dapat dijalankan dengan baik (Reviewer menggunakan Node.js versi <strong>LTS 14.17.0</strong>).</li><li dir="ltr">Menggunakan bahasa pemrograman dan teknologi lain, selain JavaScript dan Node.js.</li><li dir="ltr">Menggunakan Framework Node.js <strong>selain Hapi Framework</strong>.</li><li dir="ltr"><p dir="ltr">Melakukan kecurangan seperti tindakan plagiasi.</p></li></ul><p><br /></p><h2 dir="ltr">Forum Diskusi</h2><p dir="ltr">Jika mengalami kesulitan, Anda bisa menanyakan langsung ke forum diskusi. <a href="https://www.dicoding.com/academies/261/discussions?query=&amp;query_criteria=&amp;sort=&amp;sort_direction=&amp;title=&amp;tutorial=14967&amp;keywords=&amp;creator=&amp;page=1">https://www.dicoding.com/academies/261/discussions</a>.</p><p dir="ltr"><br /></p><h2>Ketentuan Proses Review</h2><p>Beberapa hal yang perlu Anda ketahui mengenai proses <em>review</em>:</p><ul><li dir="ltr">Tim Reviewer akan mengulas submission Anda dalam waktu <strong>selambatnya 3 (tiga) hari kerja</strong> (<em>tidak termasuk Sabtu, Minggu, </em>dan<em> hari libur nasional</em>).</li><li>Karena kelas ini merupakan kelas Pemula, <strong>mekanisme </strong>proses review pada kelas ini hanya <em>pengetesan aplikasi </em>dan <em>tidak dengan kode</em>. Untuk mendapatkan hasil review yang lebih komprehensif Anda dapat mengikuti kelas selanjutnya.</li><li dir="ltr">Tidak disarankan untuk melakukan <em>submit berkali-kali</em> karena akan memperlama proses penilaian.</li><li dir="ltr"><p dir="ltr">Anda akan mendapatkan notifikasi hasil review submission via email.
